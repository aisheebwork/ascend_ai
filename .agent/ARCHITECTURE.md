# ARCHITECTURE

## Flow
```
React+Vite+Tailwind SPA (Cloudflare Pages)
  │  Google Sign-In → Firebase Auth → ID token (JWT)
  │  POST /api/analyze  (Authorization: Bearer <idToken>, body {sql})
  ▼
Cloudflare Pages Function  functions/api/analyze.ts
  1. auth.ts: verify ID token (jose + Google JWKS; iss/aud == FIREBASE_PROJECT_ID)
  2. analyzer.ts: deterministic findings (PII + partition) from bundled metadata
  3. gemini.ts: suggestion text via Gemini REST (fallback: rule-based templates)
  → JSON AnalysisResult
  ▼
SPA renders results, then writes the run to Firestore users/{uid}/analyses (client SDK)
History panel subscribes to that subcollection.
```

## Module boundaries
- **Analyzer** is pure/deterministic and unit-tested (`test/analyzer.test.ts`).
- **Gemini** layer only formats suggestion text; never affects detection.
- **Metadata** is built at compile time into `functions/_lib/metadata.bundle.json`
  (gitignored, regenerated by `npm run build:metadata`).
- Shared API types live in `functions/_lib/types.ts`; frontend re-exports via `src/types.ts`.

## The two flows (from architecture_resources)
- **Flow 1 (PII):** tables → `is_sensitive` cols + `pii_role_id` → used in filter? → suggest decrypt.
- **Flow 2 (Partition):** `require_partition_filter` + partition col not in filter → suggest filter.
